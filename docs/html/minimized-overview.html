Schema-aware AI

Describe it. Get runnable SQL.

It reads your actual tables and columns — not invented ones — and never auto-executes. Generated SQL lands in a cell for you to review first.

PLAIN ENGLISH →"Top 10 customers by revenue last 30 days, exclude cancelled orders"
SELECT c.name, SUM(o.amount) AS revenue
FROM customers c JOIN orders o ON c.id = o.customer_id
WHERE o.created_at >= NOW() - INTERVAL '30 days'
  AND o.status <> 'cancelled'
GROUP BY c.id, c.name ORDER BY revenue DESC LIMIT 10;

Schema-aware — uses your real table and column names.

Common questions

Good to know

Is PgStudio free?

Yes — completely free, no subscription or account. Install from the VS Code Marketplace or Open VSX in under a minute.

Which AI providers are supported?

OpenAI (GPT-4o), Anthropic Claude, Google Gemini, GitHub Models (free, no API key), and the VS Code Language Model API. GitHub Models requires no configuration.

How is it different from pgAdmin or DBeaver?

PgStudio runs inside VS Code — no separate app to install or switch to. Your notebooks, explorer, and AI assistant stay in the same window as your code.

What PostgreSQL versions are supported?

PostgreSQL 12 through 17, plus compatible managed services like Amazon RDS, Supabase, Neon, and other pg-compatible databases.

Does it support SSH tunnels?

Yes — built-in SSH tunnel support connects to databases on private networks or behind firewalls with no extra tooling.

Can I share notebooks with my team?

.pgsql notebooks are plain files that commit to git like source. Your team replays the same queries against their own connections — no export needed.
